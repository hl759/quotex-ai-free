ALPHA HIVE - MIGRACAO POSTGRES

O que muda:
- A IA continua com o mesmo comportamento operacional.
- O que muda e a camada de persistencia: estado, scans, journal, ledger de edge, reputacao dos especialistas e memorias adaptativas passam a poder sobreviver fora da instancia.

Backend automatico:
- Se ALPHA_HIVE_DATABASE_URL ou DATABASE_URL apontar para Postgres, a IA usa Postgres.
- Se nao houver URL de Postgres, ela continua em SQLite local.

Variaveis principais:
- ALPHA_HIVE_DATABASE_URL=postgresql://usuario:senha@host:5432/db
  ou
- DATABASE_URL=postgresql://usuario:senha@host:5432/db

Dependencia nova:
- psycopg2-binary

Migracao automatica:
- learning memory -> store persistente
- specialist reputation -> store persistente
- journal trades -> collection persistente
- edge ledger -> collection persistente
- edge snapshot -> store persistente
- scans/meta/capital state -> store persistente

Observacao importante:
- No primeiro boot com Postgres vazio, a IA tenta aproveitar os arquivos locais existentes como fonte de migracao.
- Depois disso, o Postgres vira a fonte principal.

Impacto no app:
- Interface: igual
- Inteligencia: igual
- Persistencia: mais forte
- Risco de amnesia apos restart/redeploy: muito menor
